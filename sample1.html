Implementation

How Hygge works under the hood

Database

Hygge scrapes data from Google, TripAdvisor, and Wikipedia to build its initial database of popular destinations

This gives Hygge basic information about sites in each city as a foundation for recommendations.

Persona Matching

Users build a Persona on registration — Hygge uses euclidean distance to calculate match scores

The Persona captures preferences and interests, which Hygge uses to generate personalized recommendations.

Machine Learning

Hygge uses ML to continuously update scoring for both database and user profiles

When users aren't sure what they want, Hygge learns from their behavior over time to improve recommendations.

Key Takeaways

Personalization and tangibility are the next frontier for digital experiences

  • Personalization is the future — as ML matures, fewer inputs should yield more relevant results
  • The challenge: how to get the most relevant data with the fewest user inputs
  • Digital convenience can strip away the emotional experience — happy memories deserve a tangible form too
  • Recommendation accuracy requires more data research and domain expertise
  • The interface and flow need continuous iteration to be truly streamlined
